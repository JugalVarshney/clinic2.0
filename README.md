# Clinic Management System 2.0

## Overview
This is a modernized version of the clinic management system, rebuilt using Python/Django for the backend with plans for a React frontend. The goal is to upgrade from the legacy PHP system to a more maintainable, secure, and scalable solution.

## Current Progress

### Backend (Django)
✅ **Environment Setup**
- Python 3.11.9 virtual environment created
- Django 5.2.14 and Django REST Framework 3.17.1 installed

✅ **Project Structure**
- Django project: `clinicbackend`
- Django app: `clinicapp`

✅ **Database Models** (`clinicapp/models.py`)
- **User**: Extended Django's AbstractUser with role-based fields (doctor/patient/admin)
- **Doctor**: One-to-one relationship with User, includes specialization, experience, fees
- **Drug**: Simple model for medication tracking
- **Prescription**: Links doctors to patients with prescription details

✅ **Configuration** (`clinicbackend/settings.py`)
- Added `rest_framework` and `clinicapp` to INSTALLED_APPS
- Configured MySQL database connection to match original schema
- Kept DEBUG=True for development

### Database Schema Mapping
The Django models map to the original MySQL tables:
- `users` → User model
- `doctors` → Doctor model (with OneToOne to User)
- `drugs` → Drug model
- `prescriptions` → Prescription model
